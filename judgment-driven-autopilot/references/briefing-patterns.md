# Briefing Patterns

## Default Closeout

```text
Decision: handled / conditional / blocked.

Changed: <one or two concrete artifacts>.
Verified: <checks and outcome>.
Remaining: <highest severity blockers only>.
Next: <best next cycle>.
```

## When The User Is Frustrated

```text
Short answer: <plain truth>.

What was actually done: <evidence>.
What still will not work: <hard blockers>.
What can work if we do it right: <conditions>.
Next move: <one concrete action>.
```

## When Work Was Large

```text
Completed: <scope>.
Evidence: <validators / screenshots / runs>.
Important finding: <one thing the user would not see from logs>.
Blocked/risky: <ordered list>.
Recommended next cycle: <specific>.
```

## What To Avoid

- Do not end with a vague "if you want".
- Do not paste raw logs unless requested.
- Do not list every file when only two matter.
- Do not say "fully done" when a required gate did not run.
- Do not bury a hard blocker below polish notes.
