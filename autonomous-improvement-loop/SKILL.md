# Autonomous Improvement Loop

## Compatibility Alias

This skill name is preserved for old prompts. It is no longer the canonical autonomous loop.

When triggered, route the work to [critical-autopilot](../critical-autopilot/SKILL.md) and use Critical Autopilot's loop, evidence discipline, validation ladder, and briefing style.

## Preserved Emphasis

Carry these priorities into the Critical Autopilot cycle:

- Prefer action over questions and treat the user as the final evaluator.
- Infer hidden goals such as submission readiness, operational safety, usability, and skill reusability.
- Use gap mapping and triage: blocker, concern, polish; fix-now, queue, intentional, or human decision.
- Use adjacent specialist skills when they directly apply.
- When improving a skill, apply the loop recursively and produce a concise evolution backlog.

For detailed behavior, use `critical-autopilot/references/absorbed-patterns.md`, `critical-autopilot/references/gap-lenses.md`, and `critical-autopilot/references/skill-self-audit.md`.
